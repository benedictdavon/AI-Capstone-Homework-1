# Personal Finance Expense Prediction ML

A reproducible Python ML mini-project for predicting daily personal expenses and discovering spending-pattern clusters from synthetic student finance data. The project now generates a multi-person dataset with spending personas, trains supervised models from the command line, runs person-level clustering, and writes metrics and figures to `results/`.

This repository began as **AI Capstone Homework 1**. It has been upgraded from a notebook-style report into a small command-line project suitable for portfolio review.

The negative R2 is intentionally reported rather than hidden. That is a useful limitation of the mini-project and a clear target for future feature work.

## Problem Statement

Personal finance behavior is shaped by routines, income timing, commuting patterns, food preferences, weekend activity, and irregular purchases. This project asks:

1. Can a supervised model predict each person's daily expense total from calendar and recent-spending features?
2. Can unsupervised learning recover meaningful spending profiles from generated behavior?
3. How much does synthetic-data design affect model performance and cluster interpretation?

## Dataset Generation

The dataset is fully synthetic. No real banking or personal financial data is used.

`scripts/generate_data.py` creates transaction-level data for about 50 people over one year. Each person is assigned one profile type:

- `frugal_student`
- `food_heavy_spender`
- `commuter`
- `social_weekend_spender`
- `irregular_high_variance_spender`
- `part_time_worker`
- `lifestyle_gym_spender`

Generated transaction fields include:

- `person_id`
- `profile_type`
- `date`
- `category`
- `transaction_type`
- `amount_ntd`
- `day_of_week`
- `is_weekend`
- `month`
- `payday_distance`
- `income_source`
- `payment_method`
- `description`

Raw generated transactions are written to:

```text
data/raw/synthetic_expense_transactions.csv
```

Processed daily features are written to:

```text
data/processed/daily_person_features.csv
```

## Methods

### Supervised Learning

The supervised task predicts `daily_expense_ntd` for each person-day.

The pipeline aggregates transactions to daily person-level rows and builds features including:

- calendar features: `day_of_week`, `is_weekend`, `month`, `payday_distance`
- recent behavior: `lag1_expense_ntd`, `rolling7_expense_ntd`
- profile indicators
- person indicators

The model is a `RandomForestRegressor` evaluated with a time-based holdout split and `TimeSeriesSplit` cross-validation. The script compares four feature sets:

- `calendar_only`
- `calendar_lag`
- `calendar_rolling`
- `full_behavioral`

Outputs:

- `results/metrics/supervised_metrics.csv`
- `results/tables/feature_importance.csv`
- `results/tables/supervised_predictions.csv`
- `results/figures/supervised_rmse_by_feature_set.png`
- `results/figures/feature_importance_full_behavioral.png`

### Clustering / Unsupervised Learning

The clustering pipeline summarizes each person into spending behavior features, including average daily expense, volatility, weekend lift, income level, and recent-spending statistics.

It evaluates K-Means for k=2 through k=10 using:

- inertia
- silhouette score
- Davies-Bouldin Index
- Calinski-Harabasz score

The selected k is chosen by highest silhouette score, tie-broken by lowest Davies-Bouldin score. A Gaussian Mixture Model with the selected k is also fit to provide soft cluster assignments and confidence scores.

Outputs:

- `results/metrics/clustering_metrics.csv`
- `results/tables/cluster_assignments.csv`
- `results/figures/clustering_metrics.png`
- `results/figures/spending_clusters_pca.png`

## Setup

```bash
git clone https://github.com/benedictdavon/AI-Capstone-Homework-1.git
cd AI-Capstone-Homework-1
python -m venv .venv
```

Activate the environment:

```bash
# macOS/Linux
source .venv/bin/activate

# Windows PowerShell
.\.venv\Scripts\Activate.ps1
```

Install dependencies:

```bash
python -m pip install --upgrade pip
pip install -r requirements.txt
```

## Run Commands

Run the full pipeline:

```bash
python scripts/run_all.py
```

Run individual steps:

```bash
python scripts/generate_data.py --num-people 50 --seed 42
python scripts/run_supervised.py
python scripts/run_clustering.py
```

If the raw transaction file is missing, the modeling scripts can generate it automatically:

```bash
python scripts/run_supervised.py --generate-if-missing
python scripts/run_clustering.py --generate-if-missing
```

Run tests:

```bash
pytest
```

## Limitations

- The dataset is synthetic and rule-based, so results do not validate real-world financial prediction performance.
- The profiles are hand-designed and may make clustering easier than it would be on real bank data.
- Irregular purchases create large daily spikes that are hard to predict from simple lag and calendar features.
- The supervised model currently predicts daily total expense, not category-level expense.
- The pipeline is intentionally small and does not include experiment tracking, model registry, or deployment.
- Original notebooks remain in the repository as exploration history, but they are no longer the primary workflow.

## Future Work

- Save a compact model artifact for inference examples.
